Network Layers on Client and Server-

Question7- Explain the function of the TCP/IP model and itslayers.
Answer- 
Function of TCP/IP Model-
Enables communication between devices on a network
Breaks data into smaller packets for transmission
Ensures reliable delivery of data
Reassembles data at the destination
Provides addressing and routing of data

Layers of TCP/IP Model-
The TCP/IP model has four layers:

1. Application Layer
Provides network services to applications
Interfaces directly with user applications

Examples:
HTTP (web browsing)
FTP (file transfer)

2. Transport Layer
Responsible for end-to-end communication
Ensures reliable data transfer

Protocols:
TCP (reliable, connection-oriented)
UDP (fast, connectionless)

3. Internet Layer
Handles logical addressing and routing
Determines the best path for data

Protocol:
IP (Internet Protocol)

4. Network Access Layer
Deals with physical transmission of data
Includes hardware and network media

Types of Internet Connections-

Question9- How does broadband differ from fiber-optic internet?
Answer- 
Broadband and fiber-optic internet are two commonly used types of internet connections. They differ mainly in technology, speed, reliability, and performance.

1. Technology
Broadband uses copper wires (telephone lines or cable TV lines) to transmit data using electrical signals.
Fiber-optic internet uses optical fibers and transmits data as light signals.

2. Speed
Broadband provides moderate internet speeds, usually sufficient for basic browsing and streaming.
Fiber-optic internet offers very high speeds, often reaching up to gigabits per second (Gbps).

3. Reliability
Broadband connections can be affected by distance and network congestion, leading to slower speeds at times.
Fiber-optic connections are more reliable and provide consistent performance.

4. Latency
Broadband has higher latency, which may cause delays in online activities.
Fiber has very low latency, making it ideal for gaming and video conferencing.

5. Availability
Broadband is widely available in most areas.
Fiber-optic internet is limited to certain regions, mainly urban areas.

6. Cost
Broadband is generally cheaper and more affordable.
Fiber-optic internet can be more expensive due to advanced technology.

Used primarily for system programming

Improved portability and consistency

3. Modern Enhancements

Further updates like C99, C11, and C18 added new features

Improved safety and functionality

Importance of C Programming
1. Foundation for Other Languages

Many languages like C++, Java, and Python are influenced by C

2. System Programming

Used to develop operating systems and compilers

3. High Performance

4. Portability

Programs can run on different systems with minimal changes

Why C is Still Used Today
1. Embedded Systems

Widely used in microcontrollers and embedded devices

5. Control Flow Statements in C-

Question5- Explain decision-making statements in C (if, else, nested if-else, switch). Provide examples of each.
Answer- 
Introduction-

In C, decision-making statements allow a program to make choices based on conditions. They control the flow of execution.

1. if Statement
Explanation

Executes a block of code if a condition is true

Syntax
if (condition) {
    // code
}
Example
int a = 10;
if (a > 5) {
    printf("a is greater than 5");
}
2. if-else Statement
Explanation

Executes one block if condition is true, otherwise another block

Syntax
if (condition) {
    // true block
} else {
    // false block
}
Example
int a = 3;
if (a > 5) {
    printf("Greater");
} else {
    printf("Smaller");
}
3. Nested if-else
Explanation

An if-else inside another if-else

Used for multiple conditions

Example

Question15- Explain recursion in C++ with an example
Answer- 
Recursion in C++ is a technique where a function calls itself to solve a problem. It is useful for problems that can be broken down into smaller subproblems.

Definition

A recursive function is a function that calls itself repeatedly until a certain condition (called the base case) is met.

Key Components of Recursion
1. Base Case

The condition where the recursion stops

Prevents infinite looping

2. Recursive Case

The part where the function calls itself with a smaller or simpler input

Example: Factorial of a Number

The factorial of a number n is:
        n!=n×(n−1)!

Program
#include <iostream>
using namespace std;

int factorial(int n) {
    if (n == 0) {        // Base case
        return 1;
    } else {
        return n * factorial(n - 1);  // Recursive call
    }
}

int main() {
    int num = 5;
    cout << "Factorial = " << factorial(num);
    return 0;
}
Output
Factorial = 120

Constraints are conditions or rules enforced on data columns to maintain valid and consistent data in a database.

1. NOT NULL Constraint
Ensures that a column cannot have NULL (empty) values

Combines NOT NULL + UNIQUE

);

Ensures that values satisfy a specific condition

Example:

);

6. DEFAULT Constraint
Assigns a default value if no value is provided

Example:

    city VARCHAR(50) DEFAULT 'Ahmedabad'

11. SQL Joins-

Question24- Explain the concept of JOIN in SQL. What is the difference between INNER JOIN, LEFTJOIN, RIGHT JOIN, and FULL OUTER JOIN?
Answer-
Introduction-
A JOIN in SQL is used to combine data from two or more tables based on a related column (usually a common key). It helps retrieve meaningful data from multiple tables.

Definition-
A JOIN is an SQL operation that links tables using a common field to produce a combined result.

Types of JOINs in SQL
1. INNER JOIN
Definition
Returns only the rows that have matching values in both tables.

Example
SELECT *
FROM Students
INNER JOIN Orders
ON Students.id = Orders.student_id;

2. LEFT JOIN (LEFT OUTER JOIN)
Definition
Returns all rows from the left table and matching rows from the right table.
If no match, NULL values are returned for the right table.

Example
SELECT *
FROM Students
LEFT JOIN Orders
ON Students.id = Orders.student_id;

3. RIGHT JOIN (RIGHT OUTER JOIN)
Definition
Returns all rows from the right table and matching rows from the left table.
If no match, NULL values are returned for the left table.

Example
SELECT *
FROM Students
RIGHT JOIN Orders
ON Students.id = Orders.student_id;

4. FULL OUTER JOIN
Definition
Returns all rows from both tables.
If there is no match, NULL values are filled for missing data.

Example
SELECT *
FROM Students
FULL OUTER JOIN Orders
ON Students.id = Orders.student_id;

Key Differences-
JOIN Type                                   	Description	                                                                     Result
INNER JOIN	                    Returns only matching rows from both tables           	                                   Common data only
LEFT JOIN (LEFT OUTER JOIN)	    Returns all rows from the left table and matching rows from the right table	              Left table + matches
RIGHT JOIN (RIGHT OUTER JOIN)  	Returns all rows from the right table and matching rows from the left table	             Right table + matches
FULL OUTER JOIN               	Returns all rows from both tables                                                 	All data (matched + unmatched)

19. Rollback and Commit Savepoint-

Question40- Explain the concept of SAVEPOINT in transaction management. How do ROLLBACK and COMMIT interact with savepoints?
Answer- 
Definition
A SAVEPOINT is a point within a transaction to which you can partially roll back instead of undoing the entire transaction.

Purpose of SAVEPOINT-
Allows partial rollback
Helps manage complex transactions
Prevents loss of all changes

Syntax
SAVEPOINT savepoint_name;

Example
BEGIN;

INSERT INTO Students VALUES (1, 'John');
SAVEPOINT sp1;

INSERT INTO Students VALUES (2, 'Alice');
SAVEPOINT sp2;

ROLLBACK TO sp1;

👉 The second insert is undone, but the first remains

Interaction with ROLLBACK and COMMIT-
1. ROLLBACK with SAVEPOINT
Purpose
Used to undo changes up to a specific savepoint

Syntax
ROLLBACK TO savepoint_name;

Effect
Only changes after the savepoint are undone
Changes before the savepoint remain intact

2. COMMIT with SAVEPOINT-

Purpose
Saves all changes permanently

Effect
All savepoints are removed after COMMIT
Changes cannot be rolled back after commit

Question41- When is it useful to use savepointsin a database transaction?
Answer-
Situations Where SAVEPOINT is Useful-
1. Complex Transactions
When a transaction involves multiple steps
Helps undo only specific parts without affecting the whole transaction

2. Error Handling
If an error occurs at a certain step
You can roll back to the last savepoint instead of restarting everything

3. Testing Intermediate Steps
Useful during testing or debugging
Allows checking results at different stages

4. Large Data Operations
While inserting or updating large amounts of data
Prevents loss of all progress if something fails

5. Maintaining Data Consistency
Ensures partial data changes can be corrected
Helps maintain database integrity

6. Multi-Step Business Processes

Example: Banking transaction (debit → credit)

If one step fails, rollback only that part

Example
BEGIN;

INSERT INTO Accounts VALUES (1, 1000);
SAVEPOINT sp1;

UPDATE Accounts SET balance = balance - 500 WHERE id = 1;
SAVEPOINT sp2;

ROLLBACK TO sp1;
